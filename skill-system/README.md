O programa foi organizado utilizando o padrão básico de arquitetura de Porgamação orienteda objetos, com getter, setter, contructor, e destructors para classes e subclasses

para as diferentes habilidades foi aplicado uma classe pai abstrata. cada habilidade diferente herdam dessa classe e fazem o uso da caracteriística de polismorfismo para o seu funcionamento

o foco principal da modelagem do software foi a escalabilidade e manutenabilidade. a fragmentação da lógica torna mais facil de adicionar e consetar pedaços do código.

até o momento o código foi contruido com foco em atender os requisitos minimos, mas há espaço para adição de uma simpels e criativa gameplay que o torna algo além de um sistema de controle de habilidades de personagem
